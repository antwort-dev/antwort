# Cluster Validation Harness

Validates Antwort against real ROSA HCP clusters with live LLM backends.

## Prerequisites

- ROSA HCP cluster with Antwort deployed
- vLLM InferenceService serving a model
- Network access to cluster routes

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `CLUSTER_ANTWORT_URL` | Yes | - | Antwort route URL (e.g., `https://antwort.apps.cluster.example.com/v1`) |
| `CLUSTER_VLLM_URL` | No | - | Direct vLLM URL for baseline comparison |
| `CLUSTER_API_KEY` | No | - | API key for authenticated requests |
| `CLUSTER_MODEL` | Yes | - | Model name (e.g., `Qwen/Qwen2.5-7B-Instruct`) |
| `CLUSTER_TIMEOUT` | No | `120` | Per-test timeout in seconds |
| `CLUSTER_ANTWORT_VERSION` | No | `unknown` | Antwort version (git commit or tag) |
| `CLUSTER_BFCL_ALL` | No | `false` | Set to `true` to run full BFCL suite (~4,700 cases) |

## Running

```bash
# Run all cluster tests
make cluster-test

# Run specific test category
go test -tags cluster ./test/cluster/ -run TestBasic -v

# Run BFCL benchmarks only
make cluster-test-bfcl

# Run with orchestrator (interactive model selection)
test/cluster/run.sh
```

## Test Categories

| Category | Test Pattern | Description |
|----------|-------------|-------------|
| Basic inference | `TestBasic*` | Non-streaming and streaming requests |
| Multi-provider | `TestMultiProvider*` | Compare Antwort vs direct vLLM |
| Tool calling | `TestToolCall*` | Function calling with real models |
| BFCL benchmark | `TestBFCL*` | Berkeley Function Calling Leaderboard subset |
| Background mode | `TestBackground*` | Async request processing |
| RAG | `TestRAG*` | File search and citations |
| Auth | `TestAuth*` | Authentication validation |
| Conversations | `TestConversation*` | Response chaining |

## Results

Results are written to `test/cluster/results/`:
- `raw/` contains JSON summaries per run
- Markdown reports generated by `report.sh`
- `latest.json` for documentation site integration
