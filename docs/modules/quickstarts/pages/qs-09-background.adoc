= Quickstart 09: Background Workers
:description: Deploy antwort with separate gateway and worker components for asynchronous background processing.

Deploy antwort with separate gateway and worker components for asynchronous background processing.
The gateway accepts HTTP requests and queues background work.
Workers poll for queued requests and process them independently.
Both share a PostgreSQL database.

*Time to deploy*: 10 minutes (after LLM backend is running)

== Architecture

This quickstart deploys three services:

* *antwort-gateway* - HTTP server that accepts client requests and queues background work
* *antwort-worker* - Background processor that polls for queued requests (no HTTP port)
* *PostgreSQL* - Shared storage for response state and the background queue

[source]
----
Client --> Gateway (HTTP) --> PostgreSQL <-- Worker (poll)
----

The gateway and worker run the same binary with different `--mode` flags.
They scale independently: add more gateway replicas for HTTP throughput, more worker replicas for background capacity.

== Prerequisites

* xref:index.adoc[Shared LLM Backend] deployed and running
* `kubectl` or `oc` CLI configured
* PostgreSQL instance accessible from the cluster (or use the bundled StatefulSet)

== Deploy

[source,bash]
----
# Create namespace
kubectl create namespace antwort

# Deploy all components (PostgreSQL, gateway, worker)
kubectl apply -k quickstarts/09-background/base/ -n antwort

# Wait for PostgreSQL
kubectl rollout status statefulset/postgres -n antwort --timeout=120s

# Wait for gateway and worker
kubectl rollout status deployment/antwort-gateway -n antwort --timeout=60s
kubectl rollout status deployment/antwort-worker -n antwort --timeout=60s
----

== Test

=== Submit a Background Request

[source,bash]
----
kubectl port-forward -n antwort svc/antwort-gateway 8080:8080 &
export URL=http://localhost:8080

# Submit a background request
RESPONSE=$(curl -s -X POST "$URL/v1/responses" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "/mnt/models",
    "input": [
      {
        "type": "message",
        "role": "user",
        "content": [{"type": "input_text", "text": "Explain quantum computing in detail."}]
      }
    ],
    "background": true
  }')

RESPONSE_ID=$(echo $RESPONSE | jq -r '.id')
echo "Queued: $RESPONSE_ID (status: $(echo $RESPONSE | jq -r '.status'))"
----

=== Poll for Completion

[source,bash]
----
while true; do
  STATUS=$(curl -s "$URL/v1/responses/$RESPONSE_ID" | jq -r '.status')
  echo "Status: $STATUS"
  [ "$STATUS" = "completed" ] || [ "$STATUS" = "failed" ] && break
  sleep 2
done

# Get the result
curl -s "$URL/v1/responses/$RESPONSE_ID" | jq '{status: .status, output: .output[0].content[0].text[:100]}'
----

=== Monitor the Queue

[source,bash]
----
# Check queue depth
curl -s "$URL/v1/responses?status=queued&background=true" | jq '.data | length'

# List in-progress requests
curl -s "$URL/v1/responses?status=in_progress&background=true" | jq '.data[] | {id: .id}'
----

== Scaling

[source,bash]
----
# Scale gateway for HTTP throughput
kubectl scale deployment antwort-gateway -n antwort --replicas=3

# Scale workers for background capacity
kubectl scale deployment antwort-worker -n antwort --replicas=8
----

Gateway and worker Deployments are independent.
Scaling one does not affect the other.

== Configuration

=== Gateway (`config-gateway.yaml`)

[source,yaml]
----
engine:
  mode: gateway
  backend_url: http://llm-predictor.llm-serving.svc.cluster.local:8080
  default_model: /mnt/models

storage:
  type: postgres
  postgres:
    dsn: postgres://antwort:antwort@postgres:5432/antwort?sslmode=disable
    migrate_on_start: true

server:
  port: 8080
----

=== Worker (`config-worker.yaml`)

[source,yaml]
----
engine:
  mode: worker
  backend_url: http://llm-predictor.llm-serving.svc.cluster.local:8080
  default_model: /mnt/models
  background:
    poll_interval: 5s
    drain_timeout: 30s
    staleness_timeout: 10m
    heartbeat_interval: 30s
    ttl: 24h

storage:
  type: postgres
  postgres:
    dsn: postgres://antwort:antwort@postgres:5432/antwort?sslmode=disable
----

| 1
| Background processor. Polls PostgreSQL for queued requests. No HTTP port.

| 1
